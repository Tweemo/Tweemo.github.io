What are the differences between relative, absolute, and fixed positioning?

Document flow

Before I get into the difference between relative, absolute and fixed positioning, I need to explain what document flow is. Document flow is the arrangement of page elements based off CSS positioning and the order of HTML.

Document flow will normally follow the order in which the elements are entered into HTML. This order will depend on what elements are used, inline elements will be ordered left to right and block elements will be ordered top to bottom. CSS positioning allows the elements in this flow to be moved and different positioning can affect the flow differently.

For our examples we will be using boxes to represent elements.

In this example, our elements are block elements which explains the top-down order.

Relative Positioning

Relative positioning is a type of positioning that moves the desired element in relation to its original location. It is important to be aware that relative position does not remove the element from the document flow and thus other elements will not fill the empty space.

In this example, the element we want to move is the green box. The location at where this green box ends up will be based on where it originally was. The original location is shown by the blue outline.

The position of the green box is as follows:

position: relative;
left: 60px;
top: 20px;

With the code we can interpret that the left edge of our green box is now 60px from the original location. The same can be said about the top edge as it is 20px from where the top edge was.

Absolute Positioning

Absolute positioning is a type of positioning that shifts the element in relation to the nearest containing element with a position. When there are two elements that have positions they are considered the 'parent' and the 'child' elements.

When we give a position for the 'child' element it will use the side of the 'parent' and compare it to the same 'child' side. However, when there is no 'parent' or the 'parent' element is static, the 'parent' becomes the viewport/browser.

Elements with absolute positioning ARE removed from the document flow. This means other elements will fill in the space no longer occupied by the 'child' element.

In this example, there are no 'parents' thus the 'parent' becomes the viewport/browser. By using the code:

position: absolute;
top:100px;
left: 30px;

We can see that the box has moved 100px from the top and 30px from the left.

Fixed Positioning

Fixed positioning is a type of positioning based on the viewport (browser), this means the 'parent' of the element will be viewport. The way you would move elements with fixed positioning will be similar to how the example above.

However, bear in mind that fixed will stay on top of your content and will stay in the same position even if you scroll. The two positioning types are affected by scrolling, which means the element will not follow you when you scroll.

Elements with fixed positioning will move when the viewport is resized and it will sit infront of other content. Since fixed positioning is always relative to the viewport, it will stay in the same position even if you scroll down the page.
